!DOCTYPE html is the tag at the top of a document that declares the document type and the language we're using (html: hypertext markup language).

html is the tag that wraps around the entirety of the document. Lang defines the language we are using (en-us: English, United States), and dir defines which direction the text should be read from (ltr: left to right).

head is the tag that contains the document's title, meta data, and links that need to be loaded at the beginning of the document. This is where you would link any appropriate stylesheets.

title is the tag that declares the title of the document in which you're working. When viewing the document in a browser, it shows up as the name of tab.

meta charset is the meta tag that declares the appropriate character set for the document to draw upon. Currently the most popular charset is utf-8 (utf: unicode transformation format). This matters for unicode and HTML codes, which can be used in place of characters not commonly found on keyboards. Examples of these can be found here. More information about what utf-8 is and how it's different from other unicode transformational formats can be found here.

viewport is the meta tag that tells your computer how the window should be viewed. width=device-width and initial-scale=1 both say that the width of the web page should be the width of the device currently hosting that page. This allows you to resize the browser window and makes your website more responsive.

body is the tag that contains the body of your webpage. This is where the meat of the content is. This is everything that you will see on the page.

header is the tag that hosts content that lives at the top of your page. This is where you would put your page's title and navigation (if you have one).

h1 is one of six tags used for titles, h1 being the biggest and h6 being the smallest. They're header tags in that they're used as headers within the content of your document.

p is your paragraph tag. Every time you need to start a new paragraph, p is the tag you would use.

div is a container tag. It can be used to group sections of your code. Another example of a container tag is span.

br is a tag for breaks in your text. It is usually used in between paragraph tags, and is unique in that it doesn't require a closing tag.

b and strong are both tags used to make text bold. The difference between the two is that screen readers will pronounce words within strong tags with extra emphasis, while words within b tags won't be. In the case of a piece of text, bolded words (such as titles) would use the b tag, while bolded words within the text itself (such as those used in dialogue) would use the strong tag.

i and em are both tags used to make text appear in italics. The difference between the two is that screen readers will pronounce words within em tags with extra emphasis, while words within i tags won't be. Again, in the case of a piece of text, italicized words (such as titles) would use the i tag, while italicized words in dialogue would use the em tag. It all depends on whether or not the italicization is used for the sake of verbal emphasis or not.

u is the tag used for underlining things. One thing to note here is that underlined pieces of text can be confused for links, so use these sparingly.
